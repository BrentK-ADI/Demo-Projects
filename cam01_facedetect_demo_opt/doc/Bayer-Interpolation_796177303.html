Bayer Interpolation

The second algorithm under review is the demosaicing of the Bayer patterned image date.  This algorithm converts the Bayer format into RGB for use the CNN, and always user friendly for viewing on a display or saving to a file.  There are several interpolation algorithms and techniques that have been devised. The simplest and most common is via bi-linear interpolation, which is utilized here.

Bilinear Interpolation Algorithm

Revisiting the overview of the Bayer data format, recall the rows and columns feature interleaved data in a fixed pattern.  In order to capture the red, green and blue components for a given pixel, the center pixel and its 8 neighbors are required.

Integrated Rotation

Traditionally a when interpolating Bayer images, the image is read out row first and reconstructed in the same orientation as the source image.  As you may recall, this project requires the image be rotated 90 degrees to match the orientation of the trained model.  To accomplish this, the Bayer interpolation will be performed column first, with the column data loaded into the rows of the CNN.  This approach has no affect on the interpolation math described above, however will drive some of the optimization and implementation decisions going forward.
