Bayer: Iteration 9

The final iteration of the Bayer algorithm focuses on utilizing the SIMD parallel instructions to further expand the efficiency of the math operations.  As with AWB, the instructions being utilized are specific to the Cortex-M4 processor, and steers the code base to be incompatible with other architectures.  

SIMD Instructions

In addition to the instructions utilized as part of the AWB optimization, additional SIMD instructions will be introduced to support the Bayer.

pkhbt and pkhtb are packing instructions which creates a 32-bit word based on 16-bit components of two different 32-bit words.  The TB and BT prefix represent Top-Bottom/Bottom-Top which specify the order at which the source register components are read in, and each supports rotation capabilities to align the data.  We'll be using several variations of these packing instructions including pkhbt, pkhbt_lsl16, and pkhtb_asr16.

uhadd8 takes two 32-bits of 4x 8-bit unsigned numbers, adds them, then right shifts by 1 (halving the result) and storing the data to a new 32-bit vector. Essentially, this is in parallel, taking the average between 4 pairs of 8-bit unsigned numbers.

usub8 performs 4 8-bit unsigned subtractions in parallel

And finally ror performs a right rotation.  This itself is not a SIMD instruction, however a C native call to rotate right does not exist, so this will be using it is assembly form.

Algorithm Updates

FetchBayerValues

The first update is to the FetchBayerValues.  In the baseline example, when working across word boundaries, a set of shifts was used along with an OR operation to correctly pack the word for use.  This can be accomplished in fewer steps using the PKHBT and ROR instructions.  The ROR is required due to the PKHBT not supporting a word swap between values.


static inline pixel_data_t FetchBayerValues(uint32_t* srcPtr, uint32_t split)
{
    pixel_data_t row;
    row.word = *srcPtr;//Get the primary word
    if( split )
    {
        //Get the next word, and join its lower half with the upper half of the
        //first word
        row.word = __PKHBT(*(srcPtr+1), row.word); 

        //Need to rotate 16 bits to align everything in order
        row.word = __ROR_16(row.word);
    }
    return row;
}

Bayer Calculations

The need to average multiple pixels values in support of the Bayer interpolation provides an ideal location to utilize the SIMD instructions.  The movement of data for both odd and even rows is shown in the figure below.  For readability, each pixel is assigned a label 1-12, and the average of pixels is denoted by the / symbol.  While the mapping of source pixels to the destination may look chaotic, there is a large savings in processing.  Previously, for a single row (2 column calculations), 8 individual addition operations, and 4 individual shift operations were used to calculate the color channel values, not including any overhead associated with moving data between registers.  This has been reduced to 4 processor instructions.

Note, there is still one value in both the odd and even rows which use a sum and shift to calculate.  This could be accomplished with an additional rotate and UHADD8 instruction, however given only a single value will benefit from this calculation, the processing savings is likely negligible.

void bayer_bilinear_demosaicing_col_to_row(uint8_t *srcimg, uint32_t src_width, uint32_t src_height,
                                           uint32_t col, uint32_t h_offset, rgb888_pixel_t *dstimg0,
                                           rgb888_pixel_t *dstimg1, uint32_t dst_count)
{
    pixel_data_t row0, row1, row2;
    pixel_data_t vertAvg, horzAvg, tempRow, tempRow2;
    uint32_t isSplit;
    uint32_t* srcPtr32 = (uint32_t*)srcimg;
    uint32_t rowStride4 = (src_width / 4);

    rgb888_pixel_t tempPixel;

    assert((col & 1) == 0);      //Needs to be a even column
    assert((h_offset & 1) == 0); //Needs to be an even row
    assert((dst_count & 1) == 0); //Even number of dest pixels

    if( col & 0x2 )
    {
        isSplit = 1;
    }
    else
    {
        isSplit = 0;
    }
    
    //Preload Rows 2 and 1
    srcPtr32 += ((src_width * (h_offset + dst_count +1)) + col) / 4;
    row2 = FetchBayerValues(srcPtr32, col);
    srcPtr32 -= rowStride4;
    row1 = FetchBayerValues(srcPtr32, col);
    srcPtr32 -= rowStride4;

    //Loop through the image. 2 rows will be worked on per iteration
    for(int i = 0; i < dst_count; i+=2 )
    {
        //Fill in row0
        row0 = FetchBayerValues(srcPtr32, isSplit);
        srcPtr32 -= rowStride4;

        // Odd row (G R G R) - Row 0
        //         (B G B G) - Row 1
        //         (G R G R) - Row 2
        vertAvg.word = __UHADD8(row0.word, row2.word);
        tempRow.word = __PKHBT_LSL16(row1.word, vertAvg.word);
        tempRow2.word = __PKHTB_ASR16(vertAvg.word, row1.word);
        horzAvg.word = __UHADD8(tempRow.word, tempRow2.word);

        //vertAvg is the average of the columns between row0 and row 2
        //tempRow is [B[0] G[1] G[0] R[1]]
        //tempRow2 is [B[2] G[3] G[2] R[3]];
        //horzAvg is [ B[0&2] G[1&3] G_Corners[0&2] R Corners[1&3]]

        //Now we have all the calculations needed to do our operations
        tempPixel.color_data.g = row1.bytes.p1;
        tempPixel.color_data.b = horzAvg.bytes.p0;
        tempPixel.color_data.r = vertAvg.bytes.p1;
        *dstimg0++ = tempPixel;

        //Same note as above with regard to tricky assembly math
        tempPixel.color_data.g = (uint8_t)(((uint16_t)horzAvg.bytes.p1 + (uint16_t)vertAvg.bytes.p2) >> 1);
        tempPixel.color_data.b = row1.bytes.p2;
        tempPixel.color_data.r = horzAvg.bytes.p3;
        *dstimg1++ = tempPixel;
        

        //Shift the 3 row window up a row, fetching a new row 0
        row2 = row1;
        row1 = row0;
        row0 = FetchBayerValues(srcPtr32, isSplit);
        srcPtr32 -= rowStride4;


        // Even row (B G B G) - Row 0
        //          (G R G R) - Row 1
        //          (B G B G) - Row 2
        //Do some parallel math
        vertAvg.word = __UHADD8(row0.word, row2.word);
        tempRow.word = __PKHBT_LSL16(row1.word, vertAvg.word);
        tempRow2.word = __PKHTB_ASR16(vertAvg.word, row1.word);
        horzAvg.word = __UHADD8(tempRow.word, tempRow2.word);

        //vertAvg = [ Bs[0] Gs[1] Bs[2] Gs[3] ]
        //tempRow is [G[0] R[1] Bs[0] Gs[1]]
        //tempRow2 is [G[2] R[3] Bs[2] Gs[3]];
        //horzAvg is [ G[0&2] R[1&3] B_Corners[0&2] G Corners[1&3]]
        //Now we have all the calculations needed to do our operations

        //NOTE: All of these assignments can be done with some assembly to reduce
        //over head. To be continued....
        tempPixel.color_data.r = row1.bytes.p1;
        tempPixel.color_data.b = horzAvg.bytes.p2;
        //The following line can be done with some tricky assembly code. For now
        //Just do it with C for read ability.
        //Note casting is needed to ensure no overflow
        tempPixel.color_data.g = (uint8_t)(((uint16_t)horzAvg.bytes.p0 + (uint16_t)vertAvg.bytes.p1) >> 1);
        *dstimg0++ = tempPixel;

        tempPixel.color_data.g = row1.bytes.p2;
        tempPixel.color_data.b = vertAvg.bytes.p2;
        tempPixel.color_data.r = horzAvg.bytes.p1;
        *dstimg1++ = tempPixel;
        
        //Prep the next iteration
        row2 = row1;
        row1 = row0;
    }
}

Normalization

Finally, a USUB8 instruction is used to normalize the RGB888 data prior to being loaded into the CNN FIFO. This will be perform all 3 -128 operations in parallel.

        for (unsigned int y = 0; y < crop_y; y++) { // ... here.
            rgb888 = rgb888_buffer0[y];
            // Normalize from [0, 255] -> [-128, 127]
            rgb888.pixel_data = __USUB8(rgb888.pixel_data, 0x00808080);

            // Loading data into the CNN fifo
            while (((*((volatile uint32_t *)0x50000004) & 1)) != 0)
                ; // Wait for FIFO 0
            *((volatile uint32_t *)0x50000008) = rgb888.pixel_data; // Write FIFO 0
        }
